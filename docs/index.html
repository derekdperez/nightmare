generate_income.sh

This version is intentionally boring in the best way: a single static index.html file with inline CSS and JavaScript only. No backend, no fetch calls, no build system, no client-side framework router, and no server-side rendering. That is exactly the kind of site GitHub Pages is designed to host.

Why this should work on Pages

  • Single root index.html
  • No nested routes or route rewrites
  • No external build output needed
  • No server-side code at all

What changed

The earlier dynamic blog approach depended on live GitHub API requests. This version removes that dependency completely and keeps all content embedded in the page.

Deployment target

Put this file at repository root as /index.html and publish from main /(root).

Plugins

Keep plugin code small, predictable, and easy to remove.

Plugin rules of thumb

  • One clear responsibility
  • Structured input and output
  • Minimal hidden state
  • Predictable artifact paths
  • Explicit error handling

Starter skeleton

from pathlib import Path
import json

def plugin_name():
    return "example_plugin"

def run_plugin(input_payload, output_dir):
    output_path = Path(output_dir)
    output_path.mkdir(parents=True, exist_ok=True)

    result = {
        "plugin": plugin_name(),
        "urls": input_payload.get("urls", []),
    }

    (output_path / "example_plugin.json").write_text(
        json.dumps(result, indent=2),
        encoding="utf-8"
    )
    return result
