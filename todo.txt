Add this page: HTTP requests 
HTTP Requests will contain a list of ALL requests made by any worker, and when available, the response content for that request. Each request should create a row in a data table that shows the domain, requested url, request type (get, post, etc), request headers, request body the request size, response code, response size, response time, response headers, response body, and links to load the request or response contents. The text in the data table cells should contain the full content, but each column needs a max size of 15% of the viewport width, and it should have a maximum row height of 3x the default row size. After that, the cell will cut off any extra text, but the cell msut still contain it - double clicking a cell to select the entire text should highlight the full content even if it is not visible.
